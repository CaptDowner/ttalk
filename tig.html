Tig Commands

Keybindings (default)

KeyActionBrowsing state variables

d diff view. %(commit) selected commit ID
l log view. %(blob) selected blob ID
p pager view. %(branch) selected branch name
t (directory) tree view.%(stash) selected stash name

g grep view. %(file) selected file
b blame view. %(lineno) selected line number. Defaults to 0
r refs view. %(ref) reference to blame or HEAD if undefined
y stash view. %(revargs) revision arguments passed on command line
h help view %(fileargs) file arguments passed on command line
s status view %(cmdlineargs) All other options passed on the command line
c stage view %(diffargs) diff-options/TIG_DIFF_OPTS used by diff view
Scrolling%(blameargs) blame-options used by blame view
Insert Scroll one line up %(logargs) log-options used by log view
Delete Scroll one line down %(mainargs) main-options used by main view
ScrBackScroll one page up %(prompt) Prompt for the argument value

View Commands

KeyAction
q Close view, if more than one view is open: jumps back to previous view. Quits last open view. Q quits all views
Enter Depends on context *log view/main view: splits view and shows commit diff *diff view: scrolls view
Tab Switch to next view
R Reload and refresh the current view
O Maximize the current view to fill the whole display
Up moves cursor up one line. *diff view from the main view: points to previous commit in main view/updates diff view to display it
Down moves cursor down one line
, Move to parent. *tree view: switch to parent dir. *blame view: loads blame for parent commit. For merges the parent is queried
View Actions
u Update status of file. In the status view, this allows you to add an untracked file or stage changes to a file for next commit (similar to running git-add ). In the stage view, when pressing this on a diff chunk line stages only that chunk for next commit, when not on a diff chunk line all changes in the displayed diff are staged.
M Resolve unmerged file by launching git-mergetool(1). Note, to work correctly this might require some initial configuration of your preferred merge tool. See the manpage of git-mergetool(1).
! Checkout file with unstaged changes. This will reset the file to contain the content it had at last commit.
1 Stage single diff line.
@ Move to next chunk in the stage view.
] Increase the diff context.
[ Decrease the diff context.

Cursor, Search & Misc

Cursor Navigation Prompt
k Move cursor one line up. :<number> Jump to specific line number, e.g. :80.
j Move cursor one line down. :<sha> Jump to specific commit, e.g. :2f12bcc.
PgUp,-,a Move cursor one page up. :<x> Execute the corresponding key binding, e.g. :q.
PgDown, Space Move cursor one page down. :!<command> Execute system command in a pager, e.g. :!git log -p.
End Jump to last line. :<action> Execute Tig command, e.g. :edit.
Home Jump to first line. :save-display <file> Save current display to <file>
:save-options <file> Save current options to <file>
:script <file> Execute commands from <file>
:exec <flags><args>Execute command using <args> (user-defined command option flags in <flags>
Search
/Search the view (prompts for regexp)
?Search backwards (Also prompts for regexp)
nFind next match for current regexp
NFind previous match for current regexp
Misc
Q Quit
<C-L>Redraw screen
z Stop background loading (for using Tig in repo with a long history)
v Show version
o Open option menu
# Toggle line numbers on/off
D Toggle date display on/off/short/relative/local
A Toggle author display on/off/abbreviated/email/username
G Toggle revision graph visualization on/off
~ Toggle (line) graphics mode
F Toggle reference display on/off (tag and branch names)
W Toggle ignoring whitespace on/off for diffs
X Toggle commit ID display on/off

: Open prompt (allows you to a command to run)
